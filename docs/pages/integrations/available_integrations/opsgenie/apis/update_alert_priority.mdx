## Update Alert Priority

Update the priority of an existing alert in Opsgenie.

<Callout type="info">
	For Opsgenie's documentation, see [Update Alert
	Priority](https://docs.opsgenie.com/docs/alert-api#update-alert-priority).
</Callout>

### Parameters:

| Field               | Description                                                                                                                   | Required |
| ------------------- | ----------------------------------------------------------------------------------------------------------------------------- | :------: |
| **Identifier**      | Identifier of the alert.                                                                                                      |   Yes    |
| **Identifier Type** | Type of the identifier that is provided as an in-line parameter. Possible values are id, tiny and alias. Default value is id. |    No    |
| **Priority**        | Priority level of the alert. Possible values: P1, P2, P3, P4, P5.                                                             |   Yes    |

## Example Request:

<Callout type="info">
	Acknowledge alert requests are processed asynchronously, therefore valid
	requests are responded to with `HTTP status 202 - Accepted`.
</Callout>

```json
{
	"result": "Request will be processed",
	"took": 0.302,
	"requestId": "43a29c5c-3dbf-4fa4-9c26-f4f71023e120"
}
```
