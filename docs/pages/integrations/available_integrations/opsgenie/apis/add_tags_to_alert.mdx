## Add Tags to Alert

Add tags to an existing alert in Opsgenie.

<Callout type="info">
	For Opsgenie's documentation, see [Add Tags to
	Alert](https://docs.opsgenie.com/docs/alert-api#add-tags-to-alert).
</Callout>

### Parameters:

| Field               | Description                                                                                                                   | Required |
| ------------------- | ----------------------------------------------------------------------------------------------------------------------------- | :------: |
| **Identifier**      | Identifier of the alert.                                                                                                      |   Yes    |
| **Identifier Type** | Type of the identifier that is provided as an in-line parameter. Possible values are id, tiny and alias. Default value is id. |    No    |
| **Tags**            | List of tags to add to the alert. Example: ["tag1", "tag2"].                                                                  |   Yes    |
| **User**            | Display name of the request owner.                                                                                            |    No    |
| **Source**          | Display name of the request source.                                                                                           |    No    |
| **Note**            | Additional alert note to add.                                                                                                 |    No    |

`tags` field examples:

```json
{
	"tags": ["OverwriteQuietHours", "Critical"]
}
```

**Example Response:**

<Callout type="info">
	Acknowledge alert requests are processed asynchronously, therefore valid
	requests are responded to with `HTTP status 202 - Accepted`.
</Callout>

```json
{
	"result": "Request will be processed",
	"took": 0.206,
	"requestId": "43a29c5c-3dbf-4fa4-9c26-f4f71023e120"
}
```
