# Open a Conversation

Opens or resumes a direct message (`im`) or multi-person direct message (`mpim`).

Required scopes:

-   `channels:manage`
-   `groups:write`
-   `im:write`
-   `mpim:write`

<Callout type="info">
	For Slack's documentation, see
	[conversations.open](https://api.slack.com/methods/conversations.open).
</Callout>

## Parameters:

| Field          | Description                                                                                                                                       | Required |
| -------------- | ------------------------------------------------------------------------------------------------------------------------------------------------- | :------: |
| **User IDs**   | Comma-separated list of user IDs. If only one user is included, this creates a 1:1 DM. If no users are included, then a channel must be provided. |    -     |
| **Channel**    | Resume a conversation by supplying an `im` or `mpim`'s ID. Alternatively, provide a list of user IDs.                                             |    -     |
| **Return IM?** | Boolean, indicates you want the full IM channel definition in the response. Defaults to `false`.                                                  |    -     |

## Example Output:

Typical successful response:

```json
{
	"ok": true,
	"channel": {
		"id": "D069C7QFK"
	}
}
```

If `Return IM` is set to `true`:

```json
{
	"ok": true,
	"no_op": true,
	"already_open": true,
	"channel": {
		"id": "D069C7QFK",
		"created": 1460147748,
		"is_im": true,
		"is_org_shared": false,
		"user": "U069C7QF3",
		"last_read": "0000000000.000000",
		"latest": null,
		"unread_count": 0,
		"unread_count_display": 0,
		"is_open": true,
		"priority": 0
	}
}
```

Typical error response:

```json
{
	"ok": false,
	"error": "channel_not_found"
}
```
