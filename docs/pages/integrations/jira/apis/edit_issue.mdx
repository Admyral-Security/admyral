# Edit an Issue

Edit an issue.

<Callout type="info">
	For Jira's documentation, see [Edit
	Issue](https://developer.atlassian.com/cloud/jira/platform/rest/v3/api-group-issues/#api-rest-api-3-issue-issueidorkey-put).
</Callout>

## Parameters:

| Field               | Description                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                  | Required |
| ------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | :------: |
| **Issue ID or Key** | The ID or key of the issue to be assigned.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                   |   Yes    |
| **Fields**          | List of issue screen fields to update provided as a JSON object, specifying the sub-field to update and its value for each field. This field provides a straightforward option when setting a sub-field. When multiple sub-fields or other operations are required, use update. Fields included in here cannot be included in update. <br/> Example: <br/><br/> `{ "summary": "Completed orders still displaying in pending", "customfield_10010": 1, "labels": ["bugfix", "blitz_test"], "priority": "High" }`              |    -     |
| **Update**          | Some fields cannot be updated using an implicit update, i.e., using `Fields` (for example, comments). Instead, you must use explicit-verb updates provided as a JSON object. See [here](https://developer.atlassian.com/server/jira/platform/updating-an-issue-via-the-jira-rest-apis-6848604/) for more information about explicit updates using verb operations. Note that fields included in here cannot be included in fields. <br/> Example: <br/><br/> `{ "labels": [{ "add": "triaged" }, { "remove": "blocker" }] }` |    -     |
| **Notify Users**    | Whether a notification email about the issue update is sent to all watchers. To disable the notification, administer Jira or administer project permissions are required. If the user doesn't have the necessary permission the request is ignored. Default: `true`                                                                                                                                                                                                                                                          |    -     |

## Example Output:

```json
{
	"expand": "renderedFields,names,schema,operations,editmeta,changelog,versionedRepresentations",
	"fields": {
		"labels": ["phishing", "triaged"],
		"summary": "Phishing email received"
	},
	"id": "10021",
	"key": "SJ-20",
	"self": "https://example.atlassian.net/rest/api/3/issue/10021"
}
```
